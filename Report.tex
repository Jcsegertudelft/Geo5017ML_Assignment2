\section{Introduction}
Airborne LiDAR data, providing 3D point cloud data with high spatial resolution, has been collected over various countries. To make full use of these datasets, additional information needs to be derived from the individual 3D points. In particular the segmentation and classification of these point clouds would allow for better modeling and monitoring of the environment. Here we take on the task of classifying segmented point clouds into one of 5 classes based solely on the geometry of their composite points, through two different machine learning methods: Support Vector Machines (SVM) and Random Forests (RF). 

\section{Features}
For the classification we need some features based on the point geometry to classify on. We define six of these possible indicators and then select the four most promising options. The six plan
\begin{itemize}
    \setlength\itemsep{0pt}
    \item 2D point density: Number of points in the data divided by the area of the convex hull of the points projected onto the $XY$-plane.
    \item Mean planarity: Mean of the planarity of the points, determined from the point itself and its 10 closest neighbours. This is done by calculating the eigenvalues of the covariance matrix of the positions of the point, sorting these eigenvalues and calculating the planarity as: $\frac{\lambda_2 - \lambda_1}{\lambda_3}$ with $\lambda_1 \leq \lambda_2 \leq \lambda_3$.
    \item Mean linearity: Mean of the linearity of the points, determined from the point itself and its 10 closest neighbours. This is done by calculating the eigenvalues of the covariance matrix of the positions of the point, sorting these eigenvalues and calculating the linearity as: $\frac{\lambda_3 - \lambda_2}{\lambda_3}$ with the $\lambda$ coefficients in ascending order.
    \item Height above ground: The height difference between the highest point of the object and the z-value of the point with percentile 5, which estimates the ground level while discarding outliers: $z_{max} - z_{5th percentile}$
    \item 3D point density: The number of points divided by the volume of the 3D convex hull enclosing the point cloud: describes how densely the points occupy the volume of the object. $\frac{N}{V_{\text{convex hull}}}$
    \item Bounding Box Ratio: This feature measures how tall an object is in comparison to its width, helping to separate elongated vertical structures such as poles from wider objects. Ratio between the vertical extent and the maximum horizontal extent of the object is calculated: $\frac{z_{\max} - z_{\min}}{\max(x_{\max} - x_{\min},\; y_{\max} - y_{\min})}$
\end{itemize}
\subsection{Feature selection}
Prior to the this decision that that seriously influence model design, we created a training and testing split with a set rng-value to ensure repeatability. We select 30\% of each class to be part of the testing split.

For selection of the four most promising features we utilized a forward search approach based on the metric $J$ defined as:
\[
J = \frac{\mathrm{trace}(S_B)}{\mathrm{trace}(S_W)}
\]
Here $S_B$ and $S_W$ are the between class and within class scatter matrices defined as:
\begin{align*}
    S_B=\sum_{k=1}^K\frac{N_k}{N}(\mu_k-\mu)(\mu_k-\mu)^T && S_W=\sum_{k=1}^{K}\frac{N_k}{N}\Sigma_k
\end{align*}
with $\Sigma_k$ the class specific covariance matrix of the features of the data samples, $N_k$ being the number of samples in class $k$, $\mu_k$ being the mean feature vector in class $k$ and $N$ and $\mu$ being the total count and mean feature vector. $K$ is the total number of classes.

We start with an empty set of features. We then add calculate the feature that results in the highest value for $J$, were it added to the set and finally repeat this until we have found our desired four features. Based on this we select the mean planarity, mean linearity, height above ground and 3D point density as the four features our models will be based on.

Using the four selected features, two classifiers from the Scikit-learn library were implemented in Python, Random Forests (RF) and Support Vector Machine (SVM). For RF the hyperparameters were varied one by one using a 5:5 train/test split. The evaluation metric used is the sklearn accuracy score defined as the fraction of correct predictions over $n_{\text{samples}}$:

\begin{equation}
\mathrm{accuracy}(y, \hat{y}) = \frac{1}{n_{\text{samples}}} \sum_{i=0}^{n_{\text{samples}} - 1} \mathbf{1}(\hat{y}_i = y_i)
\end{equation}

with $\hat{y}_i$ the predicted value of the $i$-th sample and $y_i$ the corresponding true value.

The accuracy was calculated 10 times per configuration, with the average accuracy for each configuration shown in \autoref{tab:RF_hyperparameters_results}. When a higher average accuracy was obtained by varying a hyperparameter, this setting was retained. Using this method, the most optimal parameter set was estimated.

\section{Conclusions}
The point cloud objects were classified into five categories with the RF and SVM classifiers using four features. Both classifiers performed well with an accuracy of 98\% for the SVM and 96.9\% for the RF on a 8:2 train test split. Hyperparameter tuning resulted in a small increase in accuracy, with the default parameters performing well. For RF, only switching from "gini" to "entropy" and increasing  \texttt{n\_estimators} to 200 were improvements. For SVM, the linear kernel with C=1.0 performed best, indicating the classes are linearly separable. The learning curves showed a clear ascending line over the lower training fractions, which means the small size of the dataset is the bottleneck here. While most classes were very well classified, there was confusion between poles and trees, especially with the RF classifier.

For improvement the pole/tree confusion could be solved by features that distinguish between poles that are narrow throughout and trees with a wide canopy, for example the density ratio of the upper half versus the lower half. By using features that capture multiple statistics per object, it becomes possible to identify these subtle differences between classes. A limiting factor here was the small dataset of 500 objects, that resulted in only 80 samples per class for training using a high train test split. Both better generalization and less variance between runs could be achieved with a larger dataset. 

For more representative results on unseen data a stricter separation between the training and testing set should be maintained, perhaps further splitting the training set to obtain a evaluation set or utilizing k-fold cross evaluation when optimizing the hyperparameters, as they are now partly adjusted for the testing set.
